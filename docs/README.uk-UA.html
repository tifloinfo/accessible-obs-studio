Accessible OBS Studio 1.0.1

Accessible OBS Studio — плагін доступності для 64-розрядної OBS Studio 32 у Windows 10 і 11. Він призначений для незрячих користувачів клавіатури й програм зчитування з екрана та перевірений із JAWS і NVDA. Ключ OpenAI API й Інтернет потрібні лише для функцій OpenAI.

Встановлення

Встановіть 64-розрядну OBS Studio 32.0 або новішу, а потім запустіть AccessibleOBSStudio-1.0.1-Setup.exe. Якщо OBS Studio не встановлено, пошкоджено або її версія нижча за 32.0, програма встановлення запропонує відкрити офіційну сторінку завантаження OBS і завершить роботу без змін. Старішу версію також можна оновити командою «Довідка > Перевірити оновлення» в OBS Studio. OBS Studio 32.x підтримується. За наявності OBS Studio 33 або новішої програма попередить про можливу несумісність і запропонує сторінку останньої версії плагіна, перш ніж дозволити явно вибране встановлення попри попередження. Якщо OBS Studio запущено, програма попросить повністю закрити її й натиснути «Повторити»; автоматично OBS не закривається. Плагін встановлюється до C:\ProgramData\obs-studio\plugins\accessible-obs-studio. Відсутні компоненти Microsoft WebView2 і Visual C++ додаються лише після цих перевірок; файли OBS і Qt не замінюються. На завершальній сторінці прапорець Відкрити файл ReadMe у веббраузері відкриває українську HTML-документацію.

Стандартні сполучення клавіш

  • F3: короткий опис полотна, не більше 80 символів.
  • Shift+F3: докладний опис.
  • Alt+F3: читання видимого тексту без перекладу й коментарів.
  • Ctrl+F3: опис людей і тла.
  • F4: Візуальна перевірка трансляції або запису на проблеми компонування, камери, освітлення, кадрування, різкості, зернистості, вигляду, одягу, тла й небажаних предметів.
  • Ctrl+M: фокус на видимих елементах керування медіапрогравачем.
  • F5, F7, F8: почати або зупинити трансляцію, запис чи віртуальну камеру.
  • F6 / Shift+F6: наступна / попередня основна ділянка.
  • Ctrl+0–Ctrl+5: полотно, сцени, джерела, аудіомікшер, переходи або керування.
  • Ctrl+` (клавіша під Escape): відкрити доступну консоль гучності.

Команда Accessible OBS Studio: Відкрити редактор сполучень клавіш безпосередньо відкриває той самий редактор і не має стандартного призначення. Її внутрішній ідентифікатор не змінено, тому наявне призначення зберігається.

Типово Accessible OBS Studio примусово дозволяє всі сполучення клавіш OBS лише тоді, коли OBS є активною програмою. Параметр Налаштування > Додатково > Поведінка сполучень клавіш за зміни фокуса підтримується в стані Вимикати сполучення клавіш, коли головне вікно не у фокусі та відновлюється після змін. Щоб повернути керування OBS, позначте й збережіть у редакторі параметр Дозволити OBS Studio керувати роботою сполучень клавіш поза OBS. Після цього плагін припинить втручатися.

Під час першого запуску й після зміни профілю плагін порівнює заплановані призначення з наявними. Діалог з’являється лише за справжнього конфлікту. Можна вилучити тільки конфліктні призначення й застосувати стандартні сполучення Accessibility або зберегти наявні призначення; тоді конфліктні стандартні сполучення залишаться непризначеними. Параметр Більше не запитувати для цієї версії діє в усіх профілях, але скидається новою версією чи збіркою.

Редактор сполучень клавіш

Відкрийте Знаряддя > Accessible OBS Studio, щоб безпосередньо відкрити редактор сполучень клавіш. Стрілки переміщують списком команд, Tab — між вибраною командою, типово знятим прапорцем керування сполученнями OBS і кнопками. Enter або Додати чи змінити відкриває діалог призначення. Enter і OK одразу перевіряють дублікати. У разі конфлікту називається інша команда: Ні повертає до введення, Так перепризначає сполучення. Для налаштування OpenAI натисніть у редакторі кнопку Налаштування API OpenAI.

Мікшер і медіапрогравач

Ctrl+3 фокусує звичайний мікшер OBS. Плагін більше не нумерує його регулятори й не встановлює на них глобальний фільтр подій. Ctrl+` відкриває модальну доступну консоль: стрілки ліворуч і праворуч вибирають джерело, вгору й вниз змінюють гучність на 1 дБ, Home встановлює 0 дБ, пробіл перемикає вимкнення звуку, цифри 1–0 вибирають перші десять джерел.

Під час відкриття консоль отримує виняткове керування доступними на той момент джерелами, рівнями гучності та станами вимкнення звуку. Зміни негайно застосовуються в OBS, але протягом сеансу консоль не відстежує зміни через звичайний мікшер, зовнішні контролери, перемикання сцен або інші способи керування. Не змінюйте мікшер іншим способом, доки консоль відкрита; закрийте й знову відкрийте її, щоб завантажити зовнішні зміни.

Опис полотна

Усі п’ять режимів захоплюють відтворене полотно OBS. Лише найновіша відповідь має агресивну роль ARIA alert; запитання користувача ніколи не повторюється. В усіх режимах доступні уточнювальні запитання.

У вікні короткого опису кнопка Докладний опис доступна завжди, Прочитати текст — лише за наявності тексту, Люди й тло — лише за наявності людей, а Запропоновані виправлення — лише за проблеми, яку справді можна виправити автоматично. Ці команди використовують уже надіслане зображення.

Режим Люди й тло насамперед описує видимих людей, а потім їхнє безпосереднє тло. Незв’язані елементи інтерфейсу, текст і деталі сцени пропускаються, якщо вони безпосередньо не впливають на представлення людини.

Візуальна перевірка оцінює лише зовнішній вигляд трансляції або запису. Зберігаються всі візуальні перевірки компонування OBS, порожнього захоплення, камери, освітлення, повноекранного режиму Zoom, кадрування, розмиття, можливого забруднення об’єктива, зернистості, вигляду, одягу, тла й небажаних предметів. Словесний вміст ігнорується: мова, правопис, граматика, переклад, формулювання, факти, числа, тема, тон, доречність, субтитри й підписи не вважаються проблемами. Текст може бути позначено лише як візуальний об’єкт, якщо він надто дрібний, обрізаний, розмитий, малоконтрастний, затулений або сам закриває важливі елементи зображення. Діалогове вікно чи повідомлення про помилку позначається лише тоді, коли воно закриває вміст або показує видиму проблему захоплення чи компонування, але ніколи через зміст повідомлення. Автоматичне виправлення доступне лише для фіксованого списку оборотних перетворень OBS. Перевірити знову робить новий знімок і повідомляє про покращення, погіршення, зміни й візуальні проблеми, що залишилися.

Ключ API перевіряється перед збереженням, зберігається в Диспетчері облікових даних Windows і ніколи не показується. Його вилучення потребує підтвердження й завершується повідомленням про успіх.
