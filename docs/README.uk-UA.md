# Accessible OBS Studio 1.0

[![Логотип Tiflo.Info: блакитні хвилі над назвою Tiflo.info та російським гаслом «Заплющ очі й дивись»](../assets/tiflo-info-logo.jpg)](https://tiflo.info)

Accessible OBS Studio — плагін спеціальних можливостей для OBS Studio у Windows. Його створено для незрячих користувачів, які працюють із клавіатурою та програмою екранного доступу, зокрема JAWS або NVDA.

## Вимоги та встановлення

Потрібні 64-розрядна Windows 10 або 11 і підтримувана 64-розрядна версія OBS Studio 32. Власний ключ API та підключення до Інтернету потрібні лише для функцій OpenAI.

Закрийте OBS і запустіть `AccessibleOBSStudio-1.0-Setup.exe`. Інсталятор розмістить плагін у `C:\ProgramData\obs-studio\plugins\accessible-obs-studio` і перевірить наявність Microsoft WebView2 та Visual C++. Відсутні компоненти завантажуються лише за потреби, безпосередньо із серверів Microsoft; для цього потрібен доступ до Інтернету. Файли OBS і Qt не замінюються. Для видалення скористайтеся розділом Windows «Інстальовані програми». Налаштування типово зберігаються.

## Типові клавіші

- F3 — захопити полотно й надіслати його до OpenAI для аналізу.
- F4 — перейти до видимих штатних елементів керування медіа.
- F5 — почати або зупинити трансляцію.
- F6 / Shift+F6 — наступна / попередня основна область інтерфейсу.
- F7 — почати або зупинити запис.
- F8 — увімкнути або вимкнути віртуальну камеру.

Команда **Accessible OBS Studio: Open Accessible OBS Studio** відкриває редактор сполучень клавіш. Їй можна призначити сполучення, але типово його не задано.
- Ctrl+0…Ctrl+5 — перейти до відеоперегляду, сцен, джерел, аудіомікшера, переходів або елементів керування.

Під час першого запуску плагін вимикає гарячі клавіші OBS, коли головне вікно OBS не у фокусі, але тільки якщо користувач іще ніколи не встановлював цей параметр самостійно. Подальший вибір завжди зберігається.

## Редактор сполучень клавіш

Відкрийте **Знаряддя > Accessible OBS Studio**. Знайдіть команду й натисніть Enter або кнопку **Додати чи змінити**. Налаштування OpenAI відкриваються кнопкою **API Settings** у цьому вікні. У діалозі «Hot Key» одній команді можна призначити кілька сполучень. **Додати ще одне сполучення** створює додаткове призначення. Enter підтверджує, Escape скасовує. Tab, Shift+Tab, Enter, Escape, Alt+F4, сполучення з клавішею Windows і зарезервовані системні команди не перехоплюються.

Зміни записуються в OBS лише після натискання OK у головному вікні. Delete або **Видалити** очищає всі призначення вибраної команди. Під час закриття з незбереженими змінами пропонуються «Зберегти», «Відкинути» та «Скасувати».

## Аудіомікшер і медіа

Ctrl+3 переводить фокус до аудіомікшера. Видимі регулятори гучності нумеруються в порядку відображення. Клавіші 1–9 вибирають перші дев’ять джерел, а 0 — десяте. Стрілки й інші команди залишаються штатними функціями OBS. F4 фокусує елементи керування медіа, коли OBS їх показує; плагін не підміняє вбудовані медіаклавіші OBS.

## Аналіз полотна та дії

Ключ API необов’язковий. Зберігайте його лише за потреби кнопкою **API Settings** у редакторі сполучень клавіш. У діалозі працюють Tab і Shift+Tab; натискання Enter у полі ключа активує кнопку **Зберегти ключ**. Перед зашифрованим збереженням у Диспетчері облікових даних Windows перевіряються формат ключа й автентифікація OpenAI. Без ключа блокуються тільки функції OpenAI; решта можливостей плагіна продовжує працювати.

Для F3 не потрібно спочатку фокусувати попередній перегляд. Короткий звук клацання підтверджує, що запит розпочато. Під час захоплення фокус не змінюється. Потім фокус отримує вікно результату; після його закриття відновлюється попередній елемент OBS, якщо він іще існує. Плагін не відбирає фокус в іншої програми.

Уточнювальні запитання мають безпосередньо стосуватися захопленого зображення. Текст на зображенні та повідомлення користувача вважаються недовіреними даними. Сторонні запитання відхиляються. Контекст завершується після закриття вікна або нового захоплення.

Кнопка **Запропоновані дії** відкриває обмежений список зворотних перетворень OBS. Якщо джерело не вибрано, з’являється доступний список джерел. Кожна дія має окремий прапорець і рівень ризику. До натискання **Застосувати вибране** нічого не виконується. Використовуються штатні дії OBS та його журнал скасування. Керування трансляцією, записом, звуком, обліковими даними, довільними командами й параметрами виведення заборонене.

## Аналіз сумісності

Для неперевіреної основної версії OBS попередження пропонує **Скасувати**, **Усе одно запустити** та **Аналізувати сумісність**. Аналіз поєднує локальні перевірки лише для читання з офіційними джерелами OBS і визначає ризик як низький, середній, високий або невідомий. Це не гарантія сумісності.

Успішний звіт зберігається для точної версії OBS, версії плагіна та архітектури. Надалі він відкривається без нового звернення до OpenAI. Звіт показується в доступному вікні WebView2 й автоматично копіюється до буфера обміну.

## Конфіденційність, вартість і ліцензія

Для аналізу полотна до OpenAI надсилаються полотно, мова OBS, незмінна захисна інструкція та допустимі уточнення. Для перевірки сумісності надсилаються версії, відомості про залежності й фіксований перелік можливостей, але не сцени, медіафайли чи облікові дані. Витрати API сплачує власник ключа.
